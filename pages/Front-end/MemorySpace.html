# 三种数据结构

# 栈（Stack）

栈数据结构的特点：先进后出，后进先出（LIFO）。

可以通过类比乒乓球盒子来理解栈的存取方式。

存取方式：处于盒子最顶层的乒乓球5，一定是最后被放进去的，但是可以最先被使用。而想要使用底层的乒乓球1， 就必须要将上面的4个乒乓球取出来，让乒乓球1处于盒子顶层才行。

# 堆（Heap）

堆数据结构的特点：一种树状的结构且是无序（乱序）的。

堆数据结构的存取数据方式：与书架与书非常的相似。当我们想要拿哪本书的时候只需要知道书的名字就可以在书架中找到那本书， 而不用像从乒乓球盒子里取乒乓球一样，非得将上面的所有乒乓球拿出来才能取到中间的某一个乒乓球。就好比JSON格式的数据中， 存储的 key-value 是可以无序的，因为顺序的不同并不影响到使用。

# 队列（Queue）

队列数据结构的特点：先进先出（FIFO）。

如同排队过地铁安检一样，排在队伍前面的人一定是最先过安检的人。

# 变量对象与数据类型

JavaScript的执行上下文生成后会创建一个叫做 变量对象的特殊对象，JavaScript的基础数据类型往往都保存在变量对象中。

# 基础数据类型

基础数据类型共有6种：Number、Boolean、String、Null、Undefined、Symbol。

基本数据类型的特点：

  • 基本数据类型的值是按值访问的。
  • 基本数据类型的值是不可变。
  • 基本数据类型的比较是它们的值的比较。
  • 基本数据类型的变量是存放在栈内存（Stack）中的。栈内存中包括了变量的标识符和变量的值。

# 引用数据类型

JavaScript不允许直接访问内存中的位置，即不能直接操作对象的内存空间（堆内存）。我们实际操作的是对象的引用。 引用类型的存储需要内存的栈区和堆区（堆区是指内存里的堆内存）共同完成，栈区内存保存变量标识符和指向堆内存中该对象的指针（即该对象在堆内存的地址）。

引用数据类型有一种：Object。细分的话有：Object、Array、Date、RegExp、Function等等。

引用数据类型的特点：

  • 引用数据类型的值是按引用访问的。
  • 引用数据类型的值是可变的。
  • 引用数据类型的比较是引用的比较。
  • 引用数据类型的值是保存在堆内存中的对象。
// 引用数据类型间的比较是引用的比较（就是比较两个对象的堆内存中的地址是否相同）
let arr = [1,2,3];
let arr1 = [1,2,3];

arr == arr1; // false
arr === arr1; // false

# 内存空间管理

JavaScript具有自动垃圾回收机制，会定期对那些不再使用的变量、对象所占用的内容进行释放。

# 自动垃圾收集机制原理

目前各大浏览器通常采用的垃圾回收有两种方法：标记清除和引用计数。JS 中最常见的垃圾回收方式是标记清除。

工作原理：当变量进入环境时，将此变量标记为"进入环境"。当变量离开环境时，则将其标记为"离开环境。"当被标记为"离开环境"时就会被回收内存。
