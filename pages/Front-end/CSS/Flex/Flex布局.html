# Flex基本概念

在 flex 容器中默认存在两条轴，水平主轴（main axis）和垂直的交叉轴（cross axis），这是默认的设置。 也可以通过修改使垂直方向变为主轴，水平方向变为交叉轴。

在容器中每个单元块被称为：flex item。每个项目占据的主轴空间为（main size），占据的交叉轴空间为（cross size）。 需要强调：不能先入为主认为宽度就是 main size，高度就是cross size。这个要取决于主轴的方向，若垂直方向是主轴，那么项目的高度就是main size。

# Flex容器

当设置flex布局后，子元素的float、clear、vertical-align属性将会失效。

# flex-direction

决定主轴的方向（即项目的排列方向）

# flex-wrap

默认情况下项目都排在主轴线上，使用flex-wrap可实现项目的换行。

默认不换行的情况下，子元素总长度和超出父元素长度时，子元素的长度等于父元素长度除以子元素的个数，即原有的设定的子元素宽度失效； 如果换行的情况下，子元素的设定宽度不会失效，子元素超出父元素长度时，将自上往下，从左到右的排序。

# flex-flow

为flex-direction和flex-wrap的缩写。默认值为：row nowrap。

# justify-content

定义了项目在主轴的对齐方式。

# align-items

定义了项目在交叉轴的对齐方式。

对当前行的子元素进行对齐，可对单行进行赋值，但对单个子元素无效。当父元素设置flex-wrap:wrap | wrap-reverse（允许换行）且设置了align-content属性后， 貌似align-items属性就会失效！

  • 默认值为stretch。当项目未设置高度或者设为auto，将占满整个容器的高度。
  • flex-start：交叉轴的起点对齐
  • flex-end：交叉轴的终点对齐
  • center：交叉轴的中点对齐
  • baseline：项目的第一行文字的基线对齐

# align-content(建立在主轴为水平方向时测试)

定义多根轴线的对齐方式，若项目只有一根轴线，则该属性不起作用。

  • 当flex-wrap设置为nowrap时，容器仅存在一根轴线，因为项目不会换行，就不会产生多条轴线。
  • 当flex-wrap设置为wrap | wrap-reverse时，容器可能会出现多条轴线，此时就需要去设置多条轴线之间的对齐方式了。
  • flex-start：轴线全部在交叉轴上的起点对齐。
  • flex-end：轴线全部在交叉轴上的终点对齐。
  • center：轴线全部在交叉轴上的中间对齐。
  • space-between：轴线两端对齐，之间的间隔相等，即剩余空间等分成间隙。
  • space-around：每个轴线两侧的间隔相等，所以轴线之间的间隔比轴线与边缘的间隔大一倍。

# align-items与align-content之间的区别

align-items控制行内子元素对齐，而align-content控制行对齐。

# Flex项目属性

# order

定义项目在容器中的排列顺序，数值越小，排列越靠前，默认值为0。

# flex-basis（单位为px或者%）

定义了在分配多余空间之前，项目占据的主轴空间，浏览器根据这个属性，计算主轴是否有多余空间。

详细说明：覆盖当前子元素设定的宽度（width）,重新定义当前元素的宽度，前提是定义的长度值（flex-basis值）不能比当前字符串加起来的长度值小。

.item {
    flex-basis: auto;
}

默认值为：auto,即项目本身的大小，此时的item宽高取决于自身width或height的值。 当主轴为水平方向时，设置了flex-basis， 项目的宽度设置值会失效，flex-basis需要跟flex-grow和flex-shrink配合使用才能发挥效果。

  • 当 flex-basis 值为 0 % 时，是把该项目视为零尺寸的，故即使声明该尺寸为 140px，也并没有什么用。
  • 当 flex-basis 值为 auto 时，则跟根据尺寸的设定值(假如为 100px)，则这 100px 不会纳入剩余空间。

# flex-grow

定义项目放大比例。记住一点：父容器有剩余空间，该属性才能起效。

默认值为0，即如果存在剩余空间，也不放大。当所有的项目以 flex-basis 的值进行排列后，仍有剩余空间，此时flex-grow就会发挥作用。 若所有项目的flex-grow属性都为1，则它们将等分剩余空间。若一个项目的flex-grow属性为2，其他项目都为1，则前者占据的剩余空间将比其他项多一倍。 若所有项以flex-basis的值排列完后发现空间不够了，且flex-wrap: nowrap时，此时flex-grow不起作用。

# flex-shrink

定义项目的缩小比例

默认值为1，即空间不足该项目将缩小，负值对该属性无效。

  • 如果所有项目的flex-shrink属性都为1，当空间不足时，都将等比例缩小。
  • 若一个项目的flex-shrink属性为0，其他项目都为1，则空间不足时，前者不缩小。

# flex: flex-grow,flex-shrink和flex-basis的简写

flex的默认值是以上三个属性值的组合。假设以上三个属性同样取默认值，则flex的默认值是 0 1 auto。

    flex: 1;

    flex-shrink: 1;
    flex-basis: 0%;

# 容器flex-wrap与子项flex-shrink、flex-grow之间的关系

  • 当flex-wrap为wrap | wrap-reverse，且子项宽度和不及父容器宽度时，flex-grow会起作用，子项会根据flex-grow设定的值放大。
  • 当flex-wrap为wrap | wrap-reverse，且子项宽度和超过父容器宽度时，首先一定会换行，换行后，每一行的右端都可能会有剩余空间（最后一行包含的子项可能 比前几行少，所以剩余空间可能会更大），这时flex-grow会起作用，若当前所有子项的flex-grow都为0，则剩余空间保留；若当前行存在一个子项的flex-grow不为0， 则剩余空间会被flex-grow不为0的子项占据。
  • 当flex-wrap为nowrap，且子项宽度和不及父容器宽度时，flex-grow会起作用，子项会根据flex-grow设定的值放大（为0的项不放大）。
  • 当flex-wrap为nowrap，且子项宽度和超过父容器宽度时，flex-shrink会起作用，子项会根据flex-shrink设定的值进行缩小。这里有个特殊情况， 当这一行所有子项flex-shrink都为0时，即所有子项都不能缩小，会出现讨厌的横向滚动条。
  • 针对同一个子元素，flex-shrink和flex-grow只有一个能起作用。空间足够时，flex-grow起作用；而空间不足时，flex-shrink起作用。当flex-wrap值为wrap | wrap-reverse时， 表明可以换行，既然可以换行，一般情况下空间总是够的，所以flex-shrink就不会起作用。

# align-self

允许单个项目有与其他项目不一样的对齐方式

单个项目覆盖align-items定义的属性。默认值为auto，表示继承父元素的align-items属性，若没有父元素，则等同于stretch。 这个属性跟align-items属性的值是一样的，只不过align-self是对单个项目生效的，而align-items是对容器下的所有项目生效的。
