# try...catch

将能引发异常的代码块放到 try 中，并对应一个响应，然后有异常会被捕获。

只能捕获同步代码的异常。

try {
    // 模拟一段可能有错误的代码
    console.log(x+y);
} catch(e){
    // 捕获到try中代码块的错误得到一个错误对象e，进行处理分析
    report(e)
}

# window.onerror

当JavaScript运行时错误（包括语法错误）发生时，window会触发一个ErrorEvent接口的error事件，并执行window.onerror()。

语法错误会导致当前代码块运行终止，从而导致整个程序运行中断。

/*
 * @description 运行时错误处理器
 * @param {string} message 错误信息
 * @param {string} source 发生错误的脚本URL
 * @param {number} lineno 发生错误的行号
 * @param {number} colno 发生错误的列号
 * @param {object} error Error对象
 */
window.onerror = function(msg, url, lineNo, columnNo, error) {

}

# 几个异常监控的问题

  • 如何保证大家提交的代码是符合预期的？如何了解前端项目的运行是否正常，是否存在错误？代码质量体系控制和错误监控以及性能分析；
  • 若使用了脚本代码压缩，又不想将sourcemap文件发布到线上，该如何捕获到错误的具体信息？
  • CSS文件中也存在引用资源，@font-face，background-image... 等这些请求错误该如何进行错误捕获？
  • 如果用户使用网页、发现白屏，联系上我们，我们应该如何排查呢？（以用户访问页面的过程为顺序来排查：）
    • 用户没打开网络
    • DNS域名劫持
    • http劫持
    • cdn或者其他资源文件访问出错
    • 服务器错误
    • 前端代码错误
    • 前端兼容性问题
    • 用户操作出错
